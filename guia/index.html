O que é

Um pipeline completo de dublagem, rodando local

O Dublar pega um vídeo em um idioma e devolve o mesmo vídeo dublado em outro, sem depender de serviço externo pago — só Python, FFmpeg e os modelos de IA baixados na primeira execução.

🎙️ Transcrição com Whisper

Usa faster-whisper pra transcrever o áudio original com timestamps, cobrindo mais de 90 idiomas de origem.

🌐 Tradução com M2M100

Traduz os segmentos com facebook/m2m100_418M, suportando 100 idiomas de destino.

🔊 Voz com Bark ou Coqui TTS

Sintetiza a fala traduzida escolhendo entre Bark (multilíngue) ou Coqui TTS, com sincronização smart/fit/pad pra não desalinhar do vídeo.

Como funciona

Do vídeo original ao vídeo dublado, em 7 etapas

Cada execução de dublar.py passa pelo mesmo pipeline, gravando os arquivos intermediários em dub_work/ e o resultado final em dublado/.

Extração de áudio (FFmpeg)→ Transcrição (Whisper)→ Tradução (M2M100)→ Síntese de voz (Bark/Coqui)→ Sincronização (smart/fit/pad)→ Concatenação→ Mixagem final (FFmpeg)
Pré-requisitos

O que precisa estar instalado

O Dublar roda 100% local — sem chaves de API. Só precisa do Python, FFmpeg e (opcionalmente) uma GPU NVIDIA pra acelerar.

Python 3.10+

Ambiente virtual dedicado, pra isolar as dependências de IA.

# criar e ativar venv
python3 -m venv venv
source venv/bin/activate

FFmpeg

Obrigatório — sem ele nada funciona (extração de áudio e mixagem final).

# Ubuntu/Debian
sudo apt install ffmpeg

Dependências Python

Torch (CPU por padrão) + libs do projeto listadas em requirements.txt.

# instalar deps
pip install torch --index-url https://download.pytorch.org/whl/cpu
pip install -r requirements.txt
Guia de uso · passo a passo

Da instalação ao primeiro vídeo dublado

Fluxo básico em Linux. Pra Windows, use instalar.bat e dublar.bat video.mp4 (veja o README do repo).

1

Clone o repositório

Baixe o código e entre na pasta do projeto.

git clone https://github.com/inematds/dublar.git  # baixa o repo
cd dublar
2

Crie o ambiente e instale as dependências

Ambiente virtual Python isolado, com Torch em versão CPU (troque pela build CUDA se tiver GPU NVIDIA — veja INSTALACAO_GPU.md).

python3 -m venv venv
source venv/bin/activate
pip install torch --index-url https://download.pytorch.org/whl/cpu
pip install -r requirements.txt
3

Rode a dublagem com o script base

O dublar.py é a versão estável — transcreve, traduz (inglês → português por padrão) e sintetiza a voz com Bark, usando sincronização smart.

python3 dublar.py video.mp4 --src_lang en --tgt_lang pt --tts bark --sync smart  # gera dublado/video_dublado.mp4
4

Escolha idioma, engine de voz e sincronização

Os parâmetros principais controlam idioma de origem/destino, engine TTS e como o áudio dublado se ajusta ao tempo do vídeo.

# espanhol -> inglês, engine coqui
python3 dublar.py video.mp4 --src_lang es --tgt_lang en --tts coqui

# voz específica do Bark
python3 dublar.py video.mp4 --voice v2/pt_speaker_5
ParâmetroPadrãoDescrição
--src_langenIdioma de origem (en, es, fr, pt, etc.)
--tgt_langptIdioma de destino
--ttsbarkEngine de voz (bark ou coqui)
--syncsmartSincronização (none, fit, pad, smart)
--voice—Voz específica (ex: v2/pt_speaker_5)
--rate22050Taxa de amostragem do áudio final
5

Use a versão técnica pra vídeos de programação

O dublar_tech_v2.py tem um glossário de 100+ termos técnicos e preserva nomes de tecnologias em vez de traduzir (ex.: não traduz "React", "Docker").

python3 dublar_tech_v2.py codigo.mp4 --src_lang en --tgt_lang pt
6

Confira os resultados e os logs

O vídeo final fica em dublado/; os arquivos intermediários (transcrição, tradução, áudio segmentado, log completo) ficam em dub_work/ pra debug.

# log completo do processamento
cat dub_work/logs.json

# transcrição e tradução em legenda
cat dub_work/asr.srt
cat dub_work/asr_trad.srt
Roadmap

Versões e variações do pipeline

O projeto evoluiu como uma família de scripts, cada um adicionando um recurso sobre a versão base — sem quebrar as anteriores.

Base
dublar.pyVersão estável com as funcionalidades essenciais do pipeline. Ponto de partida recomendado.
Sync
dublar2.py / dublar31.pyMelhorias de sincronização e preservação de pausas/silêncios entre frases, mais natural para palestras.
Técnico
dublar_tech_v2.pyGlossário de termos técnicos, otimizado para tutoriais de programação — preserva nomes de tecnologias.
Sync v2
dublar_sync_v2.pySincronização avançada com controle fino de timing e melhor alinhamento de áudio.
GPU
Aceleração CUDASuporte opcional a GPU NVIDIA (CUDA 11.8+) pra acelerar transcrição e síntese de voz — veja INSTALACAO_GPU.md e INSTALL_LINUX.md no repo.
